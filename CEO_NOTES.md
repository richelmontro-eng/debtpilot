# CEO Notes

## Product observation log

- Do not expose implementation terms such as Supabase or PKCE to users.
- New users need guided onboarding before seeing the dashboard.
- Protected checking cushion requires plain-language explanation.
- Pay frequency should have one authoritative editable location.
- Sign Out should remain easy to access in the sidebar.
- Recommendation history should sync across devices.
- The dashboard should provide a briefing, not just raw metrics.
- Every recommendation must explain why it was generated.
- Database migrations must be verified before dependent features launch.
- Codex commits locally; GitHub Desktop performs the push.
